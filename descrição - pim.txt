sistema integrado em C# utilizando windows forms e banco de dados mysql para o seguinte cenário:

"Uma empresa de médio porte possui um setor de TI responsável por atender solicitações internas de suporte técnico. Atualmente, todas as requisições são recebidas por e-mail ou telefone, o que gera dificuldades no controle dos chamados, atrasos e falhas na priorização. A empresa deseja adotar um sistema integrado, onde os colaboradores possam registrar suas solicitações e a IA possa sugerir soluções automáticas ou encaminhar ao técnico adequado com base no histórico de chamados e complexidade do problema."


- Ao iniciar o sistema a primeira tela será de login: aonde terá campo para login e senha e opção para logar e se cadastrar. Se o usuário tentar logar e não encontrar login, exiba uma mensagem para o usuário. 

- ao clicar em cadastrar ir para tela de cadastro e cadastrar as seguintes informações: nome, CPF, telefone, endereço, e-mail, senha, confirmação de senha, botão cancelar e botão cadastrar. não permita cadastrar se todos os campos não forem preenchidos e exiba mensagem caso isso aconteça. Ao se cadastrar exiba mensagem informando que cadastrou e depois retorne ao login para autenticar. Vamos deixar um perfil de administrador cadastrado no sistema e esse perfil não pode ser excluído.

Se logar como usuário direcione para a tela de usuário aonde ele terá opções principais para registrar chamado, acompanhar chamado, informações de contato e avaliação de atendimento.
na tela registrar chamado ter os seguintes campos e todos com descrição para o usuário: título do chamado, categoria (hardware, software, periférico, rede, outros), nível de urgência (baixa, média, alta), descrição, anexos para arquivos, botão cancelar, botão enviar chamado.
não permita enviar chamado se todos os campos não forem preenchidos e exiba mensagem caso isso aconteça, anexos é opcional. exiba breve texto para o usuário no nível de urgência para informar e saber definir qual colocar.
gerar no topo da pagina protocolo, cada chamado terá um protocolo e a data.
na tela acompanhar chamado: um texto "qual chamado você deseja acompanhar?" Digite o protocolo, e o campo para o usuário digitar. botão acompanhar que irá exibir o chamado e as ações.
na tela informações de contato exibir as informações do usuário e opção para atualizar as suas informações, a senha deve ser ocultada na exibição.
na tela avaliação de atendimento: um texto qual chamado você deseja avaliar? texto Digite o protocolo, e o campo para o usuário digitar. campo para descrição, nota de atendimento(muito satisfeito, satisfeito, insatisfeito, muito insatisfeito).

se logar como administrador direcione para a tela de administrador aonde terá opções para visualizar chamados, visualizar chamados abertos, cadastrar técnico, excluir técnico, visualizar técnicos, cadastrar administrador, excluir administrador, visualizar administrador, e visualizar avaliações.
na tela cadastrar técnico: ter os campos e todos com descrição para Nome, ID(automático), CPF, Telefone, Área de especialização, Nível de experiência, e-mail corporativo, senha e confirmação de senha, os botões cadastrar técnico e o cancelar.
não permita cadastrar se todos os campos não forem preenchidos e exiba mensagem caso isso aconteça
na tela excluir: a exclusão será pelo ID do técnico. ao digitar ID, traz o nome, e-mail, ID e nível. Opção para excluir e validação para exclusão. 
na tela visualizar técnicos: exibir todos os técnicos com informações(nome, ID, e-mail corporativo, especialização, nível) opção para editar.
na tela visualizar avaliações: exibir as avaliações realizadas, caso não tenha, exibir mensagem que não há avaliações para serem exibidas.

se logar como técnico direcione para a tela de técnico aonde terá opções para visualizar chamados, visualizar chamados abertos.
na tela visualizar chamados exibe chamados finalizados, as ações e status.
na tela visualizar chamados abertos exibe o chamado registrado com todas as informações e opção para o técnico atender o chamado, ao atender ele pode escrever resposta, anexar arquivo, e finalizar o chamado.
As ações são resposta do técnico e o usuário terá na tela acompanhar chamado a opção de responder após o técnico realizar alguma ação.
